Q:1)What is use of setTimeout()?

Ans:-setTimeout() calls a passed-in function once after a specified delay,

    setTimeout(()=>{
        console.log("Hi")
    },1000)

Q:2)What is callback hell?

Callback Hell is essentially nested callbacks stacked below one another forming a pyramid structure. Every callback depends/waits for the previous callback, thereby making a pyramid structure that affects the readability and maintainability of the code.

        setTimeout(() => {
            console.log("Hey,")
            setTimeout(() => {
              console.log("have");
              setTimeout(() =>{
                console.log("a");
                setTimeout(() => {
                  console.log("nice");
                  setTimeout(() =>{
                    console.log("day");
                  },5000)
                },4000)
              },3000)
            },2000)
          },1000)

Q:3)What are promises and why do we need them?

Ans:-Promises are used to handle asynchronous operations in JavaScript.

A promise has three states:-

  • pending:-initial state, neither fulfilled nor rejected.
  • fulfilled:-meaning that the operation was completed successfully.
  • rejected:-meaning that the operation failed.
        const promise = new Promise(function(res,rej){
            let result="";
            if(result == "yes"){
              res("promise resolved");
            }
            else{
              rej("promise rejected");
            }
          })
          
          promise.then(function(data){
            console.log(data);
          }).catch(function(err){
            console.log("Error",err);
          })

Q:4)What is promise chaining

Ans:-Promise chaining is a syntax that allows you to chain together multiple asynchronous tasks in a specific order

        new Promise(function (resolve, reject) {
            setTimeout(() => resolve(2), 1000); 
          }).then(function (result) { 
            console.log(result);
            return result + 2;
          }).then(function (result) { 
            console.log(result); 
            return result + 2;
          }).then(function (result) {
            console.log(result); 
            return result + 2;
          });

Q:5)What is the purpose of async/await keywords?

The async function declaration declares an async function where the await keyword is permitted within the function body. The async and await keywords enable asynchronous, promise-based behavior to be written in a cleaner style, avoiding the need to explicitly configure promise chains.

The await operator is used to wait for a Promise and get its fulfillment value. It can only be used inside an async function or at the top level of a module.

        function resolved() {
            return new Promise(resolve => {
              setTimeout(() => {
                resolve('resolved');
              }, 2000);
            });
          }
          
          async function asyncCall() {
            console.log('calling');
            const result = await resolved();
            console.log(result);
            
          }
          
          asyncCall();

Q:7)Difference between undefined vs not defined vs NaN

Undefined:-undefined indicates that a variable has been declared but not given a value

      var a;
      console.log(a);

Not defined:-not defined indicates that a variable does not exist.

      console.log(b);
      let b=10;

NaN:-NaN (Not a Number) is a numeric data type that means an undefined value or value that cannot be represented, especially results of floating-point calculations.

      console.log(Math.sqrt(-37));
